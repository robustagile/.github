# Robust Agile Repository

This repository contains working materials related to **[Robust Agile](https://robustagile.com)**, including:

- custom GPT instructions
- prompt and skill files
- reference documents
- supporting assets for CTOx and AI consulting materials

## What Robust Agile Is

**[Robust Agile](https://robustagile.com)** is a **CTOx service from [Gehtsoft USA LLC](https://gehtsoftusa.com)**.

CTO/CIO offering is designed for founder-led companies that need technology execution to be economically sound, predictable, and governed under real constraints. 

It also includes AI advisory focused on practical fit, risk, and disciplined use rather than hype. 

Read more on CTOx and AI advisory at https://robustagile.com

Read [Nikolay Gekht, CTO of Gehtsoft](https://www.linkedin.com/in/nikolay-gekht/) articles in [Medium Blog](https://medium.com/@robust.agile)

## Purpose of This Repository

The purpose of this repository is to keep the core materials behind Robust Agile in one place, including:

- positioning and messaging references
- instructions for custom GPTs
- reusable writing guidance
- service descriptions and supporting documents
- research notes, drafts, and internal source material

## Working Principles

Materials in this repository should support the same general direction as the Robust Agile site:

- technology decisions must be tied to business outcomes
- trade-offs should be explicit
- risk should be surfaced early
- governance matters more than theater
- clarity is preferred over hype

## Notes

This repository is intended as a practical working space, not a polished publication layer.

If a document here is later published externally, the public website remains the primary source for how Robust Agile is described.
